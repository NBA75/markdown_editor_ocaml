OCaml — fiche de référence étudiant

Langage compilé, à typage statique inféré, fonctionnel (+ impératif & objet). Illustré par un éditeur Markdown web.

Architecture : point d'entrée + bibliothèque

NAVIGATEUR ──HTTP──► bin/main.ml ──appelle──► lib/ (logique pure) (textarea) (point d'entrée : Markdown_parser ─► Markdown_renderer (aperçu) ◄─────── serveur + routes) Document · File_service · Config n'a PAS la logique Recent_files · Text_search · Editor_state métier, il ORCHESTRE chaque module = .mli (contrat) + .ml (code) ▲ test/ : 53 tests (Alcotest)

Flux de données (pur)

"# Titre"
  ▼ Markdown_parser.parse
[ Heading (1, [Text "Titre"]) ]  (AST typé)
  ▼ Markdown_renderer.render
"<h1>Titre</h1>"
  • 2 fonctions pures : mêmes entrées → mêmes sorties.
  • Effets de bord (fichiers, réseau) confinés aux bords.

Filtrage de motifs

let rec render = function
  | Text s   -> escape_html s
  | Bold xs  -> "<strong>" ^ …
  | Italic xs-> "<em>" ^ …
  • Le compilateur vérifie qu'aucun cas n'est oublié.

Erreurs sans null

val read_file :
  string -> (string, error) result

match read_file p with
| Ok contenu -> …
| Error e    -> (* message clair *)
  • option = Some v | None (remplace null).
  • result = Ok | Error : l'échec est dans le type.
